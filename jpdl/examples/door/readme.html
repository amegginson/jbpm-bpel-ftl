Door

This example shows the tradeoff between implementing the behaviour of a door in plain Java and using a jPDL process. This is intended for developers to learn about the aspects that are handled better in a process versus plain programming.

This example consists of the following files:

  • src/main/java/org/jbpm/examples/doorjava/Door.java - The plain Java implementation of the door behaviour.
  • src/test/java/org/jbpm/examples/doorjava/DoorTest.java - A plain JUnit test that shows the operation of the Door class.
  • src/main/java/org/jbpm/examples/doorprocess/processdefinition.xml - The jPDL process definition file that specifies the door process.
  • src/main/java/org/jbpm/examples/doorprocess/gpd.xml - The graphical information for the jPDL process file that is used by the graphical eclipse editor.
  • src/test/java/org/jbpm/examples/doorprocess/DoorProcessTest.java - The plain JUnit test that shows how to work with the jPDL door process.

This example requires the following JBoss Rules libraries be on the runtime classpath:

  • jbpm-jpdl.jar
  • commons-logging.jar
  • junit-3.8.1.jar
